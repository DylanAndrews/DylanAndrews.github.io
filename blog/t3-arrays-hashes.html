Hashes vs. Arrays

May 15, 2015





If I were to quickly compare hashes and arrays I would say the following. Hashes are unordered data holders assigned to a variable and arrays are ordered data holders assigned to a variable. Arrays contain an ordered collection of values, and hashes contain an unordered collection of key-value pairs. This may be a bit confusing, though, so lets unpack it a bit and look at some examples.

In an array, every element can be accessed with its index number.

demo_array = [1, 2, 3, 4, 5]

print demo_array[3]

In this example, we are printing out the third index of the array(assigned to the variable demo_array), which would be 4. The reason it wouldn't print 3, which is a reasonable expectation, is that the first element gets an index of 0. The elements of a hash, which are unordered, cannot be accessed in this way.

As stated earlier, hashes contain an unordered collection of key-value pairs. This is what they look like.

my_hash = { "name" => "Dylan",

"age" => 27,

"foot_size" => 11

}

puts my_hash["age"]

In this example, "age" is the key and 27 in the value, and we are printing the value(27) by accessing the key "age". To be clear, the key is being accessed with the puts my_hash["age"] at the end. Just remember, the order of these key-value pairs is absolutely arbitrary and unimportant.

I hope all of this helps! Please let me know if you are still confused.
